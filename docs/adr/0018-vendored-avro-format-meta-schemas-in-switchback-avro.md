# 18. Vendored Avro format meta-schemas in switchback-avro

Date: 2026-06-21

## Status

Proposed

Relates to
[5. Vendored JSON Schema meta-schemas per parser crate](0005-vendored-json-schema-meta-schemas-per-parser-crate.md)

Relates to
[16. AsyncAPI parser library MVP in switchback-asyncapi](0016-asyncapi-parser-library-mvp-in-switchback-asyncapi.md)

## Context

ADR 0005 pins JSON Schema meta-schema vendoring per parser crate. AsyncAPI 3.x
multi-format payloads include Avro (`avroSchema_v1`). Avro IR is a different
grammar from JSON Schema (ADR 0008); a dedicated `switchback-avro` crate is the
parser-side shared layer sibling to `switchback-jsonschema`.

## Decision

Vend Avro **format** JSON Schemas in `switchback-avro` following ADR 0005
machinery:

| Mechanism | Location |
| --- | --- |
| `meta-schemas/` tree | `crates/switchback-avro/meta-schemas/` |
| `meta-schemas.lock.toml` | Per-file SHA-256 |
| `pub mod meta_schemas` | `MetaSchemaAsset`, `ALL`, `read()` |
| `meta_schemas_assets.rs` | Generated by `cargo xtask spec-vendor fetch --family avro --write-lock` |
| `tests/meta_schema_integrity.rs` | Lock parity |
| CI | `spec-vendor validate --family avro` in ci-post |

**Upstream sources** (asyncapi/spec-json-schemas on GitHub):

- `definitions/3.0.0/avroSchema_v1.json` (3.x line; also referenced by 3.1.0)
- `definitions/2.6.0/avroSchema_v1.json` (2.x line)
- `common/avroSchema_v1.json` if distinct from 3.0.0 revision

`switchback-asyncapi` keeps full document validation corpus (~720 files);
`switchback-avro` owns only Avro payload format schemas for IR parsing and
future Avro `--validate`.

**Crate API:** `AvroSchema` IR (`AvroRecord`, `AvroUnion`, `AvroEnum`,
`AvroArray`, `AvroMap`, `AvroFixed`, named-type refs),
`populate_avro_schema_body` → `SchemaBody` with Avro `payload_format` and
property tables.

**Publish order:** after `switchback-jsonschema`, before `switchback-asyncapi`.

## Consequences

- `xtask spec-vendor` gains `Family::Avro`.
- `switchback-asyncapi` depends on `switchback-avro` for Avro populate; does not
  re-vendor Avro format schemas.
- Additional Avro definition revisions added only when a consumer needs them.
